自律型・製造現場DXエージェント
n8nとGemini（AI）を組み合わせた、工場設備の異常対応を自動化するツールです 。
iXacsなどのIoT基盤からアラートを受け取ると、1分以内に「マニュアルに基づいた具体的な手順」をSlackへ通知します 。

このツールの技術的な特徴
はるしネーション回避設計 (RAG)
AIの一般的な知識ではなく、必ず社内の復旧マニュアル（GAS経由）を読みに行ってから回答します 。

「安全第一」の徹底
「機械の停止（ロックアウト）」や「接触禁止」などの安全指示を、
どんな回答よりも最優先で出力するようにプロンプトを制御しています 。

現場が迷わない「5分・15分ルール」
「5分以内に現場で確認すること」「15分以内にやるべき対処」を固定フォーマットで出力し、
現場の素早い判断を助けます 。


継続的な改善サイクル
エラー内容や対応履歴を記録し、後から改善に繋げられる設計にしています。

⚙ システムの仕組み
アラート受信: Webhookで異常検知データを受け取ります 。
データ取得: GASを使って、現在の設備の数値と、対応マニュアルを自動で検索します 。
手順生成: Gemini（AIエージェント）が、取得したマニュアルを根拠に手順書を作成します 。
Slack通知: 現場の作業員へ即座に内容を送信します。

📂 フォルダの中身
/workflow: n8nで使えるワークフロー本体（JSONファイル）が入っています。
/docs: 詳細な設計背景、ビジネスインパクトの試算、およびプロンプトの詳細は以下の仕様書（PDF）をご参照ください。[自律型製造DXエージェントの構築案の仕様書.pdf](https://github.com/user-attachments/files/24439123/DX.pdf)
/assets: システムの全体図や、実際にSlackに届く画面のスクリーンショットです。

🛠 使い方
インポート: workflow/ フォルダにあるJSONファイルをn8nに読み込みます。
設定: n8nの設定画面で、Gemini APIやSlackの連携キーを登録します（※セキュリティのため、コード内に直接書かないでください） 。
URL更新: お使いの環境（GASなど）に合わせて、連携先のURLを書き換えます。
